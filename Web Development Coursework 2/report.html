Accessibility

State 3 ways in which your site is accessible.

I conducted additional research and tried my best to follow the Web Content Accessibility Guidelines (WCAG 2.0) when creating an accessible website. A carousel in the index.html file was created using the Bootstrap framework. When creating the carousel, the knowledge of the carousel structure was taken from documentation found at the following link :https://getbootstrap.com/docs/4.4/components/carousel/#with-controls. As part of the carousel design, the 'aria-hidden' feature was added to the code, and was set to true. This is a very intriguing aspect of the code as a visually impaired person may come across the website. A visually impaired person often uses a screen reader, which goes through the website, and reads what is written on the site. Therefore, this feature was purposely incorporated into the code to increase accessibility for the visually impaired. Moreover, for features like buttons, it is not desirable for the screen reader to read out 'left button' or 'right button'. The aria hidden feature makes the screen reader skip over the button span so that a visually impaired person does not have to listen to the term 'button'. This ensures that the website is more accessible and friendly to people with disabilities. The span class 'sr-only' stands for screen reader only, which makes the reader read out 'previous' or 'next' rather than the whole piece of code.

The alt text for the description of images has been carefully and meticulously detailed to ensure that blind users understand what the image is about, as they cannot see the image. If an alt text is not provided, then the screen reader will only read out the word 'image' or may provide a file name. Rather than simply giving a name of the image, i tried to imagine what i might relay over the phone when describing the image. This ensured that the image description flowed with the text before and after it. An example of this can be found on the index.html webpage. There is a heading which reads out 'Download the Stunner Group Fitness App' which is then followed by buttons which have a setting of aria-hidden:"true" so that the buttons are skipped over. The text reader then goes to the images of an iPhone with a Stunner Group App on it. Rather than simply calling it 'iPhone image', i have used an alt tag of - 'An iPhone image with the Stunner Group App is shown' and 'a second image with the Stunner Group App is shown.'

I also made sure that the website is keyboard-friendly, increasing the accessibility. A site is keyboard-friendly if it can function without a mouse, as assistive technologies rely heavily on this feature. The content and links need to be accessed using the Tab key so that the user can have access to the various webpages, icons, and external links. By using the Tab key on the webpages that have been designed, the user is able to navigate between webpages with ease through the navigation links, Stunner Group Logo, buttons, icons and footer links. It is as simple as pressing the tab key and the enter key.

Usability

State 3 ways in which you have considered usability in your site.

Before undertaking this coursework, i did not fully appreciate the importance of responsiveness when designing a webpage. However, i overcame many obstacles when attempting to make various features responsive. The Bootstrap Grid is a very effective method for making components responsive and the main documentation that was used as guidance is found on the get bootstrap website at the following link: https://getbootstrap.com/docs/4.0/layout/grid/. I learnt the proper implementation of containers and rows in different scenarios. If a container is required for the full width of the screen, then a container-fluid class can be used rather than a container class. In addition, deciding upon grid breakpoints is of great importance, as the layout of components is important when switching from laptop screens to mobiles (extra large to extra small). For example, it was my desire to display three horizontally aligned cards on an extra large screen, and then two horizontally aligned cards stacked on top of one card underneath on a medium sized screen. This was done by using the class col-lg-4 so that each card takes up 4 units on a maximum of 12 columns on a large screen. Using col-md-6 meant that two cards will take up 6 units on a medium sized screen. By using col-sm-12, each card would take up the full width of the screen, which was needed for small/extra small sized screens (i.e. mobile devices). A feature which complimented the grid method was the Bootstrap styling of each component. The background of the components can be changed to the range of Bootstrap colours (primary, secondary, success, danger, light, dark, warning, info and muted). If you want to change the text colour to the Bootstrap shade of green, then you would use the class text-success. If you wanted to change the background of a component to the Bootstrap shade of red, you would use the class bg-danger. The majority of components such as the navbar, carousel, scrollspy and general divisions are styled using Bootstrap colours. I learnt that images are not naturally responsive when using Bootstrap. When creating the newproducts.html web page, i had difficulty making the images responsive when reducing the screen size. The w3 schools website clearly states that a Bootstrap class of img-responsive is the method of making an image responsive, found at the following link: https://www.w3schools.com/bootstrap/bootstrap_images.asp. After much searching on various forums, i came across a forum named 'stack overflow', found at the following link: https://stackoverflow.com/questions/34811268/no-img-responsive-in-bootstrap-4?noredirect=1. A member of the forum explained that the class img-responsive only works in Bootstrap 3 and that the name of the class has been changed to img-fluid in Bootstrap 4. By incorporating responsiveness with various bootstrap classes, i was able to increase the usability of the site.

Learning

State 3 things you had to learn/find out by yourself when creating your site, and describe how you did that (i.e. what searches you did, any new tools/techniques you learned, any changes/adaptations you made to a particular resource to make it work with your site).

In the last semester, i took a module named Introduction to Programming 1, and i created a bubble effect, where bubbles would move across a canvas. In this coursework assignment, I created a canvas in HTML, and gave it an ID of mycanvas. I then started to create the code in the events.js file, and declared the canvas variables and targeted the canvas element with the getElementByID command. I then set my context variable with canvas.getContext('2d'). The canvas height and width has been set to the window width and height. To get the current mouse position, i created an event listener with the same methods taught in the course material, and has been used to get the current mouse position, and works every time the user moves the mouse. I then created a class for the bubbles, to fill the bubble array, so that individual bubbles would have randomised values. I have used constructor functions and learnt how to use them effectively by looking at an example on w3 schools at the following URL: https://www.w3schools.com/JS/js_object_constructors.asp. In addition, each bubble needed x and y coordinates, which are independent from mouse x and mouse y coordinates. Direction x and direction y holds the number of bubbles animated along each step in the x and y axis, and ensures the bubbles will be bouncing back and forth around the canvas. Individual bubbles are drawn and are repeated via the update function and for every frame of the animation. The bubble screen position and mouse interaction is also updated on each frame. The check bubble position is a technique learnt in the module Introduction to Programming 1, which checks the current bubble position, checks the mouse position, moves the bubble accordingly, and then calls the draw method. As the bubbles bounce around the canvas, the purpose of the code is to check if the bubbles have reached the edge of the screen. If the bubble reaches the edge of the screen, then the direction is reversed through the directionX or directionY to make it animate in the opposite direction. I then also incorporated collision detection, which was also covered in the module Intro to Programming 1. To do this, i took the centre point of the mouse curser and the centre point of the bubble, and checked if the distance between the two centre points is less than a set distance. The formula of 'dx*dx +dy*dy' was taken from the forum - stack overflow at the following URL: https://stackoverflow.com/questions/4578111/calculating-distance-between-two-x-y-coordinates-in-php. The forum member wanted a similar solution for collision detection when working in the games industry, and i found it to be useful and implemented it as part of my Javascript code. A series of 'if statements' are used in the code, by initially checking if mouseX is smaller than this.x (current position). If this is true, then we know that the mouse is to the left of the bubble; and so the bubble should be pushed to the right (+ direction in the x-axis). Another 'if' statement checks if the bubble is far enough from the edge of the canvas, as we do not want to push it over the edge of the canvas. This has been done for all four sides of the screen, pushing the bubbles back before they get the chance to overflow. After these checks have been made, the other bubbles which are not colliding with the mouse are pushed along the x or y axis by adding their 'this.x' values to their direction x and direction y values. They are then drawn by calling the function this.draw(). After these steps were completed, a randomised function was created named function init (). A bubble array variable was assigned to an empty bubble array. The number of bubbles have been created under this function. At this point i faced difficulties in implementing random bubbles so i took guidance from one specific section in the example at the following URL: https://code.tutsplus.com/tutorials/generating-a-particle-system-with-javascript--net-10668. I was able to use the guidance to create a random size (between 1 and 5). The let direction x and let direction y determine the movement speed. A bubble object was then created with randomised values and the Array.push method was used as learnt in the module - Introduction to Programming 1 to push the values into the empty bubble array. The animation loop was then created with guidance from the following article: https://muffinman.io/javascript-animation-loop/. The function has been named animate, and a requestAnimationFrame for smooth animation/ update method has been called for each individual bubble with a for loop. Finally, the function init is called and so is animate.

The helper name was changed, and a switch statement was added to allow the code to switch between languages. Three different cases were added - English, French and German for the paragraph underneath the Protein Purchase heading. The second part of the template function passes in data and sets the language. I have set the language as English ('en'). However, by changing this part of the code to 'de', the text switches to German. If the code is changed to 'fr', then the text turns into French. I believe that i used Handlebars helpers effectively to showcase their functionality. However, one aspect i would conduct differently next time is to use the handlebars templates to enable a different language for every piece of text on each webpage. Therefore, by simply changing the state, the entire webpage text would change from English to German or French. This would be much more impressive than simply changing one line of text. In addition, i would create a separate handlebars.js file to keep all of the handlebars code in one place, rather than imbedding it as script tags into the html file. This will then fall in alignment with the way in which i have separated other Javascript code to their own files.
